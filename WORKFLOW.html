🔄 Engine Event Loop — ejecución barra por barra

Por cada barra, el EventLoop ejecuta exactamente esta secuencia. La clave anti-look-ahead: la señal se genera al close del bar t pero la orden se ejecuta al open del bar t+1.

sequenceDiagram participant DH as DataHandler participant Q as Queue participant EH as ExecutionHandler participant S as Strategy participant P as Portfolio rect rgb(240, 248, 255) Note over DH,P: ── Bar t ────────────────────────────────────── DH->>Q: MarketEvent(ts_t, open_t, close_t) Note over Q: EventLoop extrae el MarketEvent Q->>EH: fill_pending(open_t) Note over EH: Ejecuta _pending del bar t-1 EH->>Q: FillEvent(fill_price≈open_t, commission, slippage) Q->>P: on_fill → actualiza cash y posiciones end rect rgb(240, 255, 245) Note over DH,P: Estrategia ve close_t Q->>S: on_market(MarketEvent) S->>Q: SignalEvent(LONG/SHORT/EXIT, strength) P->>P: update_market → equity_curve.append(equity) Q->>P: on_signal(SignalEvent) → position sizing P->>Q: OrderEvent(quantity, direction, MKT) Q->>EH: on_order → agrega a _pending Note over EH: La orden espera el próximo open end rect rgb(255, 248, 240) Note over DH,P: ── Bar t+1 ── la orden se ejecuta acá ────── DH->>Q: MarketEvent(ts_{t+1}, open_{t+1}, close_{t+1}) Q->>EH: fill_pending(open_{t+1}) EH->>Q: FillEvent(fill_price≈open_{t+1}) end
Por qué el fill va al open siguiente: En mercados reales no podés ejecutar a exactamente el close. La señal se genera cuando el bar cierra; la orden entra al mercado en la apertura del día siguiente. Esto evita look-ahead bias en la ejecución.

Anatomía de cada evento

EventoCamposQuién lo emiteQuién lo consume

OrderEvent timestamp, symbol, order_type, quantity, direction Portfolio.on_signal() ExecutionHandler.on_order() → guarda en _pending
FillEvent timestamp, symbol, direction, quantity, fill_price, commission, slippage ExecutionHandler.fill_pending() Portfolio.on_fill() → actualiza cash + posiciones

Cómo SimplePortfolio calcula la cantidad

Cuando recibe un SignalEvent:

# direction = "LONG" o "SHORT"
quantity = (self.equity * 0.02) / price
# equity = cash + suma(posición × último_precio)
# position_pct = 0.02 → apuesta el 2% del equity por trade

# direction = "EXIT" → cierra toda la posición abierta
quantity = abs(self._positions[symbol])

Configurás position_pct al instanciar SimplePortfolio. Podés implementar tu propio Portfolio con Kelly, volatility targeting, etc.

📐 Mean Reversion — implementación completa

Una estrategia mean-reverting necesita dos capas: la lógica de señal (Strategy) y el wrapper de fit/predict (FitPredictEstimator) que conecta con el pipeline de validación.

1
Parámetros (la grilla)
window: barras de lookback del z-score. entry_z: umbral para entrar. exit_z: umbral para salir. El tuner elige la combinación óptima.
2
fit(IS)
Calcula media y std del IS. Congela esos parámetros. El OOS nunca toca esta estimación → sin look-ahead bias estadístico.
3
on_market(bar)
Calcula z-score usando los parámetros del IS. Si z cruza entry_z → emite LONG/SHORT. Si vuelve a exit_z → emite EXIT.
4
predict(OOS)
Instancia el EventLoop completo con la estrategia ya hidratada. Corre el loop. Retorna (returns_series, signals_series).

Archivo: strategy/mean_reversion.py

from __future__ import annotations
from queue import Queue
import polars as pl
from engine.events import MarketEvent, SignalEvent
from strategy.base import Strategy


class MeanReversionStrategy(Strategy):
    """
    Emite LONG cuando z < -entry_z (precio muy bajo → espera reversión al alza).
    Emite SHORT cuando z > entry_z  (precio muy alto → espera caída).
    Emite EXIT cuando |z| < exit_z  (precio volvió al rango normal).

    IMPORTANTE: mean y std se reciben del IS. La estrategia nunca los recalcula
    con datos del OOS. Así no hay look-ahead bias estadístico.
    """

    def __init__(
        self,
        events_queue: Queue,
        mean: float,
        std: float,
        entry_z: float = 2.0,
        exit_z:  float = 0.5,
    ) -> None:
        super().__init__(events_queue)
        self._mean    = mean
        self._std     = std
        self._entry_z = entry_z
        self._exit_z  = exit_z
        self._position: float = 0.0  # +1 long, -1 short, 0 flat
        self._signals:  list[float] = []

    def on_market(self, event: MarketEvent) -> None:
        if self._std == 0:
            self._signals.append(0.0)
            return

        z = (event.close - self._mean) / self._std  # z-score con params del IS

        if self._position == 0:
            if z < -self._entry_z:
                self._emit(event, "LONG",  -z)    # precio bajo → compramos
                self._position = 1.0
            elif z > self._entry_z:
                self._emit(event, "SHORT",  z)    # precio alto → vendemos
                self._position = -1.0
            else:
                self._signals.append(0.0)

        elif abs(z) < self._exit_z:
            self._emit(event, "EXIT", 0.0)      # precio normalizó → salimos
            self._position = 0.0
        else:
            self._signals.append(self._position)   # mantenemos posición

    def _emit(self, event: MarketEvent, direction: str, strength: float) -> None:
        self._events_queue.put(SignalEvent(
            timestamp=event.timestamp,
            symbol=event.symbol,
            direction=direction,
            strength=strength,
        ))
        self._signals.append(strength if direction == "LONG" else -strength)

    def reset(self) -> None:
        self._position = 0.0
        self._signals = []

Archivo: strategy/mean_reversion_estimator.py

Este archivo es el pegamento entre el engine y el pipeline de validación. Implementa el protocolo FitPredictEstimator.

Contrato FitPredictEstimator:
fit(is_segments: list[pl.DataFrame]) → self
predict(oos_data: pl.DataFrame) → tuple[pl.Series, pl.Series]

fit solo ve IS. predict corre el EventLoop sobre OOS con los parámetros congelados del IS. 
from __future__ import annotations
from queue import Queue
import polars as pl
from engine.data_handler import DataFrameDataHandler
from engine.portfolio    import SimplePortfolio
from engine.execution_handler import SimulatedExecutionHandler
from engine.event_loop   import EventLoop
from strategy.mean_reversion import MeanReversionStrategy


class MeanReversionEstimator:
    """
    Encapsula fit + predict para que CPCVEngine no sepa nada de la estrategia.
    Parámetros controlados por la grilla: window, entry_z, exit_z.
    Parámetros internos aprendidos del IS: _mean, _std.
    """

    def __init__(
        self,
        entry_z: float = 2.0,
        exit_z:  float = 0.5,
        initial_capital: float = 100_000,
        position_pct:    float = 0.02,
        slippage_pct:    float = 0.001,
        commission_pct:  float = 0.0006,
    ) -> None:
        self._entry_z        = entry_z
        self._exit_z         = exit_z
        self._initial_capital = initial_capital
        self._position_pct   = position_pct
        self._slippage_pct   = slippage_pct
        self._commission_pct = commission_pct
        # Aprendidos en fit():
        self._mean: float = 0.0
        self._std:  float = 1.0

    def fit(self, is_segments: list[pl.DataFrame]) -> "MeanReversionEstimator":
        # Concatena todos los bloques IS y estima la distribución del precio
        all_closes = pl.concat(is_segments)["close"]
        self._mean = float(all_closes.mean())
        self._std  = float(all_closes.std(ddof=1))
        return self

    def predict(self, oos_data: pl.DataFrame) -> tuple[pl.Series, pl.Series]:
        # Instancia todo el engine con los parámetros del IS congelados
        q            = Queue()
        data_handler = DataFrameDataHandler(q, "ASSET", oos_data)
        strategy     = MeanReversionStrategy(
            q, self._mean, self._std, self._entry_z, self._exit_z
        )
        portfolio    = SimplePortfolio(
            q, self._initial_capital, self._position_pct
        )
        execution    = SimulatedExecutionHandler(
            q,
            slippage_pct=self._slippage_pct,
            arancel_alyc_pct=self._commission_pct,
        )
        loop = EventLoop(q, data_handler, strategy, portfolio, execution)
        loop.run(close_open_positions=True)  # cierra posiciones al final del fold

        signals = pl.Series("signals", strategy._signals)
        return portfolio.returns_series, signals

¿Qué hace el tuner con la grilla? Por cada combinación de parámetros, llama al estimador en múltiples folds internos del IS. Agrega los ganadores por mediana (para numéricos) o moda (para categóricos). El resultado es el consensus_params que se usa para el fit externo.

Purging y Embargo — por qué importan

Purging (label_horizon)

El último grupo IS antes del test se trunca en label_horizon barras. Motivo: si tu etiqueta mira h barras hacia adelante, las últimas h barras del IS "ven" datos del OOS. Se eliminan.

IS: [........ ▓▓▓▓]  ← purge 1 barra
Test:              [████]
Embargo (embargo_pct)

El primer grupo IS después del test se recorta al inicio en max(1, len(grupo) × embargo_pct) barras. Motivo: autocorrelación serial entre el OOS y el IS siguiente.

Test: [████]
IS:         [▒▒ .......] ← embargo al inicio

import polars as pl
from validation.config import ValidationConfig
from validation.cpcv   import CPCVConfig, CPCVEngine
from validation.tuning import build_nested_cpcv_runner
from strategy.mean_reversion_estimator import MeanReversionEstimator

# ── 1. Datos ──────────────────────────────────────────────────────
data = pl.read_csv("activo.csv", try_parse_dates=True).sort("timestamp")
# Columnas mínimas: timestamp, open, close

# ── 2. Configuración de validación ───────────────────────────────
val_cfg = ValidationConfig(
    bars_per_year=252,        # datos diarios
    label_horizon=1,           # 1 barra de purge
    embargo_pct=0.01,          # 1% del fold como embargo
    block_bootstrap_reps=10_000,
    alpha_halflife_bars=20,
    n_trials=len(grid),        # activa DSR
)

cpcv_cfg = CPCVConfig(
    n_groups=6,               # → C(6,2)=15 backtests, φ=5 paths
    n_test_groups=2,
)

# ── 3. Grilla de hiperparámetros ─────────────────────────────────
grid = [
    {"entry_z": 1.5, "exit_z": 0.5},
    {"entry_z": 2.0, "exit_z": 0.5},
    {"entry_z": 2.5, "exit_z": 1.0},
]

def estimator_factory(params):
    return MeanReversionEstimator(
        entry_z=params["entry_z"],
        exit_z=params["exit_z"],
    )

# ── 4. Construir el runner anidado ────────────────────────────────
# Este runner hace los 3 actos: tune (Acto1), fit (Acto2), predict (Acto3)
runner = build_nested_cpcv_runner(
    val_cfg=val_cfg,
    grid=grid,
    estimator_factory=estimator_factory,
    n_inner_splits=5,
)

# ── 5. Ejecutar ───────────────────────────────────────────────────
report = CPCVEngine(val_cfg, cpcv_cfg).run(data, runner)

# ── 6. Resultado ─────────────────────────────────────────────────
print(report.summary())

# Opcional: grafico equity vs SPY
report.plot_vs_spy(
    output="equity_vs_spy.png",
    strategy_label="Mean Reversion",
    benchmark_ticker="SPY",
)
